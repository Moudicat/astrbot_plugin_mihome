# 🌟 AstrBot MiHome Control (v6.3.22 )

基于 [AstrBot](https://github.com/Soulter/AstrBot) 框架开发的米家智能家居云端控制插件。

旨在为用户提供**最安全、最直观**的米家设备控制体验。

## ✨ 核心特性

* **🔒 隔离式扫码登录**：登录流程通过独立子进程完成，避免阻塞主事件循环；认证凭证安全保存在 `plugin_data` 目录中，便于备份、迁移与卸载清理。
* **🧠 设备画像驱动 (Device Profiles)**：将用户别名、中文控制词与底层属性键解耦。插件内置常见设备的专属画像，可将“睡眠”“制冷”“童锁”等中文指令映射为对应设备可识别的底层属性和值。
* **📊 中文状态展示**：基于本地画像与云端返回结果，对设备状态项进行中文翻译与单位美化展示。像 `relative_humidity`、`pm2.5_density` 这类底层字段会尽量转换为“当前湿度: 60%”“PM2.5浓度: 12”这类更易读的文本。
* **🛡️ 柔性降级与防呆设计**：当设备未配置别名、未同步缓存、暂时离线、深度休眠或部分状态读取失败时，插件会优先返回本地设备画像与引导信息，尽可能保证“可看、可控、可排障”。

## 🚀 快速上手：从零到控制你的第一台设备
对于初次使用的用户，请严格按照以下 4 步完成设备绑定与控制：
### 步骤 1：扫码授权登录
对机器人发送 `/米家登录`。机器人会拉起独立登录进程并返回一个授权链接，点击打开后使用米家 APP 扫码完成授权。授权成功后，凭证将安全保存在本地。
### 步骤 2：获取设备 DID 并建立本地档案 (⚠️ 关键)
发送 `/刷新米家`。机器人会从云端拉取您账号下可见的设备列表，例如：
> `1. 【未配置别名】(米家空气净化器 4 Lite) [🟢] (12345678)`  
> `2. 【未配置别名】(空调插座) [🟢] (114514)`
**这一步很关键。**  
插件会在后台把设备 **DID** 与其官方名称一起缓存下来，后续 `/米家详情`、`/米家帮助`、`/米家控制` 都依赖这份本地档案。请记录下行尾括号中的那串数字（例如 `114514`），这就是设备 DID。
### 步骤 3：在 WebUI 配置好记的别名
*⚠️ 出于安全性与精确性考虑，未配置别名的设备默认不允许查看详情与远程控制。*
前往 AstrBot WebUI 的插件配置页，找到 `device_map` 选项，为设备 DID 配置一个你习惯使用的别名：
* 键 (Key) [别名]：`"卧室净化器"` 或 `"客厅 空调"`（支持带空格）
* 值 (Value) [DID]：`"114514"`
填写后保存配置，重载插件，再执行一次 `/刷新米家` 让缓存和别名映射同步生效。
### 步骤 4：开始查看与控制
此时你可以先发送探测指令，查看设备支持的能力与当前状态：
> `/米家详情 卧室净化器`
如需查看常用控制写法：
> `/米家帮助 卧室净化器`
再发送控制指令：
> `/米家控制 卧室净化器 开`  
> `/米家控制 卧室净化器 模式 睡眠`
如果设备成功响应，说明你的智能管家已经可以正常工作了。

## 🎮 指令指南
### 🔑 账号与管理
* `/米家登录`：拉起独立登录流程，获取扫码授权链接。
* `/米家登出`：清除当前登录状态与本地凭证。
* `/米家状态`：查看插件当前登录状态、共享设备异常与最近一次控制记录。
### 📡 设备查询
* `/刷新米家`：同步云端设备列表，输出带 DID 的设备总览，并刷新本地设备名称缓存。
* `/米家详情 [设备别名]`：先返回该设备的本地画像（可调属性、状态传感），再尝试补充实时状态数据。
* `/米家帮助 [设备别名]`：查看该设备常用控制方式、支持的控制属性与推荐指令示例。
### 🎛️ 设备控制
**基础电源控制（推荐优先使用）**
* `/米家控制 [设备别名] [开/关]`
    * *示例*：`/米家控制 落地扇 开`
**高级属性控制**
* `/米家控制 [设备别名] [属性] [值]`
    * *示例*：`/米家控制 空调 温度 26`
    * *示例*：`/米家控制 蒸煮锅 火力 3`
> 💡 提示：
> * 推荐优先参考 `/米家帮助 [设备别名]` 中给出的示例格式。
> * 某些设备支持的高级属性较多，且不同型号之间取值方式可能不同（数字、布尔、枚举混用）。
> * 对于尚未完全适配的新设备，你仍可尝试使用插件展示出的原生英文属性名进行高级控制透传。
> 💡 **设备画像未完全适配？欢迎提交 Issue！**  
> 如果你在使用过程中发现：
> * 详情页存在大量未汉化字段
> * 控制属性映射不准确
> * 某型号设备的值域和文案不匹配  
> 欢迎提交 Issue，帮助完善设备画像库。
>
> **建议附带以下信息：**
> 1. 设备的官方名称  
> 2. `/米家详情 [设备别名]` 的返回内容  
> 3. `/米家帮助 [设备别名]` 的返回内容  
> 4. 实际可用但当前未适配的控制方式  
>
> 例如可直接贴出：
> ```text
> ✅ 【您的设备名称】支持的高级调整属性:
> status, fault, target_time, target_temperature, left_time, auto_keep_warm, current_keep_warm, mode, recipe_id, recipe_name, recipe_sync, target_cooking_measure, turn_pot, turn_pot_config, texture, reservation_left_time, cooking_weight
> ```
> 这些信息将帮助我们在后续版本中补充更准确的中文画像与控制映射。

## ⚠️ 常见排障
| 现象 | 可能原因与建议 |
| :--- | :--- |
| **设备找不到 / 共享设备异常** | 第三方接口对共享设备的支持并不总是稳定。建议优先使用设备**主账号**扫码登录。 |
| **设备拒绝请求** | 你可能修改了只读属性，或传入了不符合设备要求的值类型 / 值范围。建议先查看 `/米家帮助 [设备别名]`。 |
| **提示需先刷新米家** | 本地缓存中还没有该设备的官方名称与画像匹配信息。请先执行一次 `/刷新米家`。 |
| **详情页只有本地画像，没有实时状态** | 设备可能离线、深度休眠，或者当前接口未成功读取状态。这种情况下通常仍可尝试发送基础控制指令。 |
| **网络异常 / 超时** | 宿主机与米家接口之间可能发生 TLS 断连、网络波动或请求受限。建议稍后重试，并检查网络环境。 |

## 🗺️ 未来更新计划 (Roadmap)


### 🌈 场景与自动化 (Smart Scenes)
* **原生/本地场景支持**：支持调用米家 App 预设场景或在插件内自定义快捷指令组（如 `/米家场景 晚安`）。

### 🛡️ 体验与稳定性增强 (UX & Stability)
* **状态快照缓存**：即使设备离线，也能显示最后一次成功读取的状态信息（如“5分钟前湿度为 60%”）。
* **控制回读验证**：下发指令后自动异步回读设备状态，确保控制真正生效。
* **更友好的错误解释器**：将枯燥的系统错误码转译为人类易懂的排障建议。

### ⌨️ 管理与效率工具 (Management Tools)
* **别名管理命令化**：脱离 WebUI，直接通过指令 `/米家绑定`、`/米家解绑` 管理设备别名。
* **档案一键导出**：支持导出当前设备的完整物模型 JSON，方便开发者调试与用户反馈。

### 🧠 深度智能化 (Intelligence)
* **自然语言模糊控制**：无需严格遵循指令格式，支持 `@机器人 把空调调到26度` 这种更自然的交互方式。
* **定时任务调度**：支持相对时间或绝对时间的控制预约（如“30分钟后关闭净化器”）。

### 🛠️ 关于Tool：
* 这是一个需要长期规划的事情，可能会做自然语言调取温度计之类的的数据，但是处于安全性考虑，大电器不会接入的，请**自行fork**。
