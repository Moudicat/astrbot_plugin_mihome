## ✨ 核心特性

### 1. 🛡️ 进程级沙盒登录
- **安全隔离**：采用独立子进程沙盒执行登录逻辑，彻底解决传统重定向 `sys.stdout` 带来的全局日志污染问题。
- **无缓冲实时流**：通过 `-u` 参数强迫子进程实时冲刷输出管道，配合异步流监听，第一时间抓取官方登录链接并推送至聊天窗口。
- **业务级校验**：内置 `ticket` 与 `dc` 参数完整性校验，确保在流式输出环境下捕获的链接 100% 完整有效，杜绝残缺 URL。
- **僵尸进程清理**：完善的进程回收机制，确保登录超时、强制登出或异常后，后台进程被物理切断。

### 2. 🧠 智能语义控制
- **别名映射**：支持在 WebUI 中通过 JSON 自定义设备别名（如：“净化器”、“客厅灯”），告别冗长难记的原始设备名。
- **自动属性识别**：得益于底层 `mijiaAPI` 封装，无需手动配置繁琐的 `siid` 或 `piid`，插件会自动识别设备支持的开关属性。
- **语义兼容**：内置动作别名库，支持“开/关”、“开启/关闭”、“on/off”、“true/false”等多种表达方式。

### 3. 📊 全方位排障体系
- **状态实时监控**：通过 `/米家状态` 指令，一眼查看到期凭证、活跃沙盒、最近一次控制错误及设备信息。
- **精准异常透传**：接入米家原生异常体系，准确区分“设备不存在”、“鉴权失效”与“云端接口异常”。

## 🚀 交互指令清单

所有指令均**仅限管理员**在**私聊**环境中使用。

| 指令 | 描述 |
| :--- | :--- |
| `/米家登录` | 拉起独立沙盒，获取小米官方扫码链接（有效期 120s）。 |
| `/刷新米家` | 拉取当前账号下所有设备的 DID、型号及在线状态。 |
| `/控制米家 [别名] [开/关]` | 对指定设备下发电源控制指令（需配置别名映射）。 |
| `/米家状态` | 查看插件运行报告、凭证状态及详细排障日志。 |
| `/米家登出` | 强制中止登录进程，并清除本地所有鉴权凭证（Auth.json）。 |

## ⚙️ 配置指南 (WebUI)

在 AstrBot 的 WebUI 管理页面中，找到本插件配置项：

### `device_map` (设备映射表)
请根据 `/刷新米家` 获取到的 `DID` 进行 JSON 配置。
- **Key**: 你在 QQ 里想称呼设备的名字
- **Value**: 设备唯一的 DID 字符串

**示例配置:**
```json
{
  "空气净化器": "123456",
  "落地电风扇": "123456",
  "卧室大空调": "123456"
}

## 🛠️ 安装说明

* **第 1 步：放置插件** 将插件文件夹整体放置于 AstrBot 根目录下的：  
    `data/plugins/astrbot_plugin_mihome`

* **第 2 步：安装依赖** 在服务器终端执行以下命令：
    ```bash
    pip install mijiaAPI
    ```

* **第 3 步：初始化授权** 重启 AstrBot，管理员在 **私聊窗口** 发送：
    > **/米家登录**

* **第 4 步：完成扫码** 点击机器人返回的链接进行扫码，手机端确认后，机器人提示“扫码授权成功”即可。

## ⚠️ 开发备注

* **环境要求**：AstrBot `>= 4.0.0`
* **平台适配**：深度优化 `aiocqhttp` (QQ) 与 `qq_official`。
* **安全保障**：
    * **零隐私存储**：不记录小米账号与密码明文。
    * **沙盒隔离**：采用 `Subprocess` 子进程隔离技术，登录副作用零污染。
    * **自动清理**：扫码超时自动销毁子进程，杜绝僵尸进程。
