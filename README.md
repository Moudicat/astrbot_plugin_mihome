# 🌟 AstrBot MiHome Control (vv6.2.7 Stable)

基于 [AstrBot](https://github.com/Soulter/AstrBot) 框架开发的米家智能家居云端控制插件。

旨在为用户提供**最安全、最直观、零污染**的米家设备控制体验。

## ✨ 核心特性

* **🔒 隔离级扫码登录**：底层通讯进程完全沙盒隔离，凭证安全存放于 `plugin_data`。
* **🌐 智能列表发现**：一键同步自有及共享设备，精准区分在线状态。
* **📡 动态属性探测**：刷新时自动嗅探在线设备当前公开的底层属性项。
* **🧠 自然语言控制解析**：
    * **兼容分词匹配**：兼容自然输入与引号输入，支持复杂设备别名（如 `/控制米家 客厅 空调 开` 或 `/控制米家 "卧室 净化器" 关`）。
    * **语义映射规范**：将人类语言（如 “制冷”、“高档”、“True”）自动规范化转译为底层可读枚举。

## 🚀 快速上手：从零到控制你的第一台设备

对于初次使用的用户，请严格按照以下 4 步完成设备绑定与控制：

### 步骤 1：扫码授权登录
对机器人发送 `/米家登录`。机器人会返回一个链接，点击打开并使用**米家 APP** 扫码授权。授权成功后，凭证将安全保存在本地。

### 步骤 2：获取设备真实 DID
发送 `/刷新米家`。机器人会从云端拉取您账号下绑定的设备列表，例如：
> `1. 【未配置别名】(米家空气净化器 Lite) [🟢] (12345678)`
> `2. 【未配置别名】(空调伴侣2) [🟢] (114514)`

请记录下括号里的这串数字（如 `114514`），这就是设备的 **DID**。

### 步骤 3：在 WebUI 配置好记的别名
前往 **AstrBot WebUI** 的插件配置页，找到 `device_map` 选项。将刚才的 DID 赋予一个你习惯呼叫的“别名”：
* **键 (Key) [别名]**：`卧室 净化器`
* **值 (Value) [DID]**：`114514`

填写完毕后，**保存配置并重载插件**。此时再发送 `/刷新米家`，你会看到设备已经认领了别名，并探测出了可调菜单！

### 步骤 4：尝试控制！
对机器人发送指令：
> `/米家控制 卧室 净化器 开`

如果听到机器“滴”的一声，恭喜你，你的智能管家正式上线！

## 🎮 指令指南

### 🔑 账号与管理
* `/米家登录`：拉起登录环境获取扫码二维码链接。
* `/米家登出`：安全销毁当前登录状态，清理本地凭证与现场。
* `/米家状态`：查看插件健康度与最近控制记录。
* `/刷新米家`：同步云端设备并探测属性。*(注：探测到的属性仅代表当前状态快照，不代表该属性一定允许远程写入)*

### 🎛️ 设备控制

**基础电源控制 (强烈推荐)**
* `/控制米家 [设备别名] [开/关]`
    * *示例*：`/控制米家 空调 开`
    * ⚠️ **注：基础电源开关经过稳定的链路优化，请优先使用此命令进行开关操作。**

**高级属性透传 (探索功能)**
* `/控制米家 [设备别名] [属性] [值]`
    * *示例*：`/控制米家 空调 温度 26`
    * *示例*：`/控制米家 净化器 模式 sleep`

## ⚠️ 常见排障 

| 现象 | 建议 |
| :--- | :--- |
| **设备找不到** | 云端可能由于安全策略拒绝了第三方 API 对“共享设备”的非所有者控制权。建议换用设备**主账号**进行扫码登录。 |
| **设备拒绝请求** | 您可能尝试修改了只读属性，或属性值的类型/格式超出该设备允许的范围。某些网关型设备（如红外空调伴侣）可能不支持拆分属性下发。 |
| **网络异常** | 您的宿主机与米家 API 发生 TLS 断连或 DNS 解析失败，请检查防火墙。 |
