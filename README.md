## ✨ 核心特性

* **🔒 隔离式扫码登录**：登录流程通过独立子进程完成，避免阻塞主事件循环；认证凭证安全保存在 `plugin_data` 目录中，便于备份、迁移与卸载清理。
* **🧠 设备画像驱动 (Device Profiles)**：将用户别名、中文控制词与底层属性键解耦。插件内置常见设备的专属画像，可将“睡眠”“制冷”“童锁”等中文指令映射为对应设备可识别的底层属性和值。
* **📊 中文状态展示**：基于本地画像与云端返回结果，对设备状态项进行中文翻译与单位美化展示。像 `relative_humidity`、`pm2.5_density` 这类底层字段会尽量转换为“当前湿度: 60%”“PM2.5浓度: 12”这类更易读的文本。
* **🛡️ 柔性降级与防呆设计**：当设备未配置别名、未同步缓存、暂时离线、深度休眠或部分状态读取失败时，插件会优先返回本地设备画像与引导信息，尽可能保证“可看、可控、可排障”。
* **🔑 基础控制与高级属性控制**：对所有设备均已适配直接开关，对于所有设备均支持通过原始属性调整值，例如直接下发temperature 26来调整为26度，其中已测试的设备已适配中文。

## 🚀 快速上手：从零到控制你的第一台设备

对于初次使用的用户，请严格按照以下 5 步完成设备绑定与控制：

### 步骤 1：扫码授权登录
对机器人发送 `/米家登录`。  
机器人会拉起独立登录进程并返回一个授权链接，点击打开后使用米家 APP 扫码完成授权。授权成功后，凭证将安全保存在本地。

---

### 步骤 2：获取设备 DID 与云端名称（⚠️ 关键）
发送 `/刷新米家`。机器人会从云端拉取您账号下可见的设备列表，例如：
> `1. 【未配置别名】(米家智能空气炸锅 5.5L 可视版) [🟢] (1234567)`  
> `2. 【未配置别名】(主卧 体脂秤) [🟢] (1234567)`  
> `3. 【未配置别名】(卧室空调) [🟢] (12345675677)`

其中：
- `【未配置别名】`：表示你还没在插件里给它取控制用别名
- 括号里的内容，例如 `米家智能空气炸锅 5.5L 可视版` / `主卧 体脂秤` / `卧室空调`：是米家云端当前显示名称
- 行尾括号中的数字，例如 `1234567`：就是设备 DID

这一步非常关键。  
插件会在后台缓存设备的基础档案（如 DID、Model型号、云端名称等），后续 `/米家详情`、`/米家帮助`、`/米家控制` 都依赖这份本地档案。  
请记录下你要控制设备对应的 DID。

---

### 步骤 3：在 WebUI 配置好记的设备别名
*⚠️ 出于安全性与精确性考虑，未配置别名的设备默认不允许查看详情与远程控制。*
前往 AstrBot WebUI 的插件配置页，找到 `device_map`，为设备 DID 配置一个你习惯使用的别名：

```json
{
  "卧室净化器": "1234567",
  "客厅 空调": "1234567",
  "空气炸锅": "4567567",
  "体脂秤": "8901234"
}
```
说明：
* 键（Key）：你希望在机器人里使用的设备别名（支持带空格别名，例如："客厅 空调"）
* 值（Value）：`/刷新米家` 输出中的设备 DID

---

### 步骤 4：为设备指定类别（若设备已在上表“完美适配”，则可忽略，否则 ⚠️ 强烈推荐配置）
接着在插件配置页填写 `device_category_map`，用于告诉插件“这台设备属于哪一类”。例如：

```json
{
  "卧室净化器": "净化器类别",
  "客厅 空调": "空调类别",
  "空气炸锅": "空气炸锅类别",
  "体脂秤": "体脂秤类别"
}
```
目前支持的类别有：
* `空调类别`
* `净化器类别`
* `风扇类别`
* `蒸煮锅类别`
* `空气炸锅类别`
* `体脂秤类别`
* `温湿度计类别`
* `扫地机类别`
* `无类别`

说明：
* 键（Key）必须与你 `device_map` 中配置的别名完全一致
* 值（Value）必须是上述合法类别之一
* 如果填写了非法类别，系统会自动按 `无类别` 处理

**为什么这一步重要？**
因为米家云端名称可能是用户自己改过的。配置设备类别后，插件就能赋予它：
* 中文化的可调属性
* 中文化的状态传感项
* 更友好的 `/米家帮助`
* 更稳定的 `/米家详情`

**💡 特别说明（模型降级机制）：**
本插件内置了**型号精确匹配** 机制。如果你设备的具体型号（Model，如 `yunmai.scales.ms103`）已经被开发者直接写进了核心字典库中，插件将**无视你在这里配置的类别**，直接采用最完美的型号专属配置。配置类别主要是为了给尚未收录专属型号的设备进行泛用性兜底。

填写后保存配置，重载插件，然后再执行一次 `/刷新米家`，让缓存与映射同步生效。

---

### 步骤 5：开始查看与控制
此时你可以先发送详情指令，查看设备能力与当前状态：
> `/米家详情 卧室净化器`
> `/米家详情 体脂秤`

如需查看常用控制写法：
> `/米家帮助 卧室净化器`

再发送控制指令：
> `/米家控制 卧室净化器 开`
> `/米家控制 卧室净化器 模式 睡眠`
> `/米家控制 扫地机 开始清扫`

如果设备成功响应，说明你的智能管家已经可以正常工作了。

## 🎮 指令指南

### 🔑 账号与管理
* `/米家登录`：拉起独立登录流程，获取扫码授权链接。
* `/米家登出`：清除当前登录状态与本地凭证。
* `/米家状态`：查看插件当前登录状态、共享设备异常与最近一次控制记录。

### 📡 设备查询
* `/刷新米家`：同步云端设备列表，输出带 DID 的设备总览，并刷新本地设备档案缓存。
* `/米家详情 [设备别名]`：查看该设备详情。
  * **优先级 1（型号精准匹配）**：若设备型号命中内置`MODEL_PROFILES`，展现最完美的专属属性与翻译。
  * **优先级 2（类别模板兜底）**：若未命中型号但配置了有效类别，返回大类通用的中文画像与状态。
  * **优先级 3（原始能力探测）**：若为 `无类别`，直接进入原始能力探测，返回设备支持的原始英文属性菜单。
* `/米家帮助 [设备别名]`：查看该设备常用控制方式、支持的控制属性与推荐指令示例。

### 🎛️ 设备控制

基础电源控制（推荐优先使用）
* `/米家控制 [设备别名] [开/关]`
  * 示例：`/米家控制 落地扇 开`

高级属性控制
* `/米家控制 [设备别名] [属性] [值]`
  * 示例：`/米家控制 空调 温度 26`

动作触发控制
* `/米家控制 [设备别名] [动作名]`
  * 示例（中文）：`/米家控制 扫地机 开始清扫`
  * 示例（英文）：`/米家控制 扫地机 start_sweep`
  * 示例（手误兼容）：`/米家控制 扫地机 start-sweep` 或 `start sweep`

> 💡 提示：
> 1. 推荐优先参考 `/米家帮助 [设备别名]` 中给出的示例格式。对于休眠类设备（如体脂秤），请在屏幕亮起时发送指令。
> 2. 对于尚未完全适配的新设备，若 `/米家详情` 已展示原生英文属性名，你仍可尝试使用这些原始属性名进行高级控制透传。

## 🎬 米家智能场景支持 (Scenes)

除了直接控制单个设备，本插件现已全面支持执行米家 APP 中的**“智能场景（手动执行）”**。你既可以通过指令手动触发，也可以在启用场景 Tool 后让大模型协助执行。！

### 📱 前置准备：在米家 APP 中创建场景
1. 打开米家 APP，进入底部的 **“智能”** 页面。
2. 点击右上角 **“+”** 创建新智能。
3. **触发条件**必须选择：**“手动执行”**。
4. **执行动作**：添加你想要的一系列设备动作（如：关灯、关空调、拉窗帘）。
5. 保存并起一个**有辨识度的名字**（例如：`晚安模式`、`离家全关`、`看电影`）。

---

### 🎮 场景控制指令

* `/米家场景列表`：同步并查看当前账号下所有可执行的云端场景。
  * 插件会列出你的 `场景名称`、`scene_id` 以及它所属的 `家庭`。
* `/米家场景 [场景名或scene_id]`：立即执行指定场景。
  * 示例：`/米家场景 晚安模式`
  * 示例：`/米家场景 123456789`
  * **⚠️ 防呆重名保护**：如果你在米家 APP 里建了多个同名场景（比如不同房间都有叫“开灯”的场景），插件为了防止误操作，会**拒绝直接执行**，并提示你改用唯一的 `scene_id`，或者建议你前往米家 APP 修改名称。

## 🤖 进阶：大模型自然语言联动 (LLM Tool)（测试版）

我们为大模型量身定制了安全、克制的 Tool（工具）调用能力，让你真正体验“自然语言操控智能家居”的快感！

### 🛡️ 安全的权限边界
为了防止大模型产生“幻觉”导致家里的电器乱动（例如误把冰箱关机或煮锅烧干），本插件**坚决不向大模型开放底层设备的高级属性与动作直接控制权**。
大模型**只能**调用你提前在米家 APP 里预设好的“场景”（Scene）。

### ⚙️ 如何启用大模型联动？
1. 前往 AstrBot WebUI 的插件配置页。
2. 找到并勾选 `enable_scene_tool`（是否启用米家场景 LLM Tool）。
3. 保存配置并重载插件。
4. 确保在聊天时**正确 @机器人（触发大模型对话）**。

### 💬 交互示例
配置完成后，你不再需要死记硬背 `/米家场景` 指令，直接用自然语言和机器人聊天即可：

> **你**：`@机器人 我准备睡觉了，帮我关一下灯。`
> **机器人**：（内部思考并静默调用 `execute_mihome_scene` 寻找合适的场景）
> **机器人**：`好的，已经为您执行了“晚安模式”场景，祝您好梦！`

> **你**：`@机器人 帮我打开影院模式。`
> **机器人**：`没问题，已触发“看电影”场景，窗帘和灯光已为您调暗。`

*(注：大模型会根据你的语境，自动去匹配你列表里最合适的场景名进行调用。如果命中同名场景，大模型也会反问你需要执行哪一个具体房间的场景。)*

## 🧠 关于“三层降级解析”与“兜底探测”

本插件采用**“具体型号专属字典 ＞ 用户指定大类模板 ＞ 原始能力探测”**的三层降级解析设计：

1. **型号**：如果你的设备型号已被内置，无论你怎么配置类别，`/米家详情` 都会精准返回该型号独有的属性。
2. **大类**：如果设备型号未被内置，只要你配置了类别（如 `净化器类别`），插件会用净化器通用的中文模板进行兜底渲染，返回：
```text
📖 【卧室净化器】本地设备画像:
✅ 可调属性: 屏幕, 开关, 提示音, 模式, 童锁, 风速
📡 状态传感: PM2.5浓度, 当前温度, 当前湿度, 滤芯剩余天数, 滤芯寿命百分比
```
3. **摸黑探测模式**：如果啥都没配（`无类别`），插件会进入能力探测模式，返回类似：
```text
⏳ 正在探测【手环】的能力菜单...
✅ 【手环】支持的高级属性:
battery_level, steps, heart_rate, ...
```
这意味着即使当前设备还没有中文模板，插件依然会尽量把它支持的原始属性列出来，方便你自行摸索。

## 💡 设备画像未完全适配？欢迎提交 Issue！

如果你在使用过程中发现：
1. 详情页存在大量未汉化字段
2. 控制属性映射不准确，或控制无效
3. 某型号设备的值域和文案不匹配

强烈欢迎提交 Issue，帮助我们将您的设备型号加入到**核心字典库 (MODEL_PROFILES)** 中。建议附带以下信息：
* /astrbot/data/plugin_data/astrbot_plugin_mihome中设备对应的.json文件**作为附件**文件
* 或者您已经知道model名称，直接在终端执行一下后全量复制在issue中

```bash
python - <<'PY'
from mijiaAPI import get_device_info
import json

model = "你的设备model"
info = get_device_info(model)
print(json.dumps(info, ensure_ascii=False, indent=2))
PY
```

这些信息将帮助我们在后续版本中为你这台设备进行“专属型号定做”。

## ⚠️ 常见排障

| 现象 | 可能原因与建议 |
| :--- | :--- |
| 设备找不到 / 共享异常 | 第三方接口对共享设备的支持并不总是稳定。建议优先使用设备主账号扫码登录。 |
| 设备拒绝请求 | 你可能修改了只读属性，或传入了不符合设备要求的值类型/范围。建议先查看 `/米家帮助 [设备别名]`。 |
| 提示需先刷新米家 | 本地缓存中还没有该设备的基础档案。请先执行一次 `/刷新米家`。 |
| 详情页只有英文菜单 | 该设备型号未内置，且未配置有效类别，或被识别为 `无类别`。请检查 `device_category_map`。 |
| 提示实况获取失败或无数据 | 设备可能离线、处于**深度休眠（如体脂秤未踩亮）**。这种情况下请唤醒设备后重试。 |
| 云端名称和别名不一致 | 这是正常现象。别名是你配置的入口名；云端名称是米家 App 中的显示名。 |
| 网络异常 / 超时 | 宿主机与米家接口之间可能发生 TLS 断连或被小米防火墙风控，建议稍后重试。 |
