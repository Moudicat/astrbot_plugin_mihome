# 🌟 AstrBot MiHome Control (v6.3.11 )

基于 [AstrBot](https://github.com/Soulter/AstrBot) 框架开发的米家智能家居云端控制插件。

旨在为用户提供**最安全、最直观、极度鲁棒**的米家设备控制体验。

## ✨ 核心特性

* **🔒 隔离级扫码登录**：底层通讯进程完全沙盒隔离，防阻塞主事件循环，凭证安全存放于 `plugin_data`。
* **🧠 千机千面物模型 (Device Profiles)**：彻底解耦用户别名与底层控制逻辑。插件内置官方设备特征图纸，自动将人类语言（如“睡眠”、“制冷”）转译为该设备专属的底层数字或枚举。
* **📊 全中文状态透传**：深度自动嗅探设备当前公开属性。自动过滤枯燥的工程级调试代码，将底层的英文传感器标识（如 `relative_humidity`）自动翻译并拼接单位为优雅的“当前湿度: 60%”。
* **🛡️ 柔性降级与防呆设计**：完善的强制拦截与缓存穿透软降级机制。当遇到未配置或未同步缓存的设备时，提供精准的交互引导，永远保障核心控制链路的高可用。

## 🚀 快速上手：从零到控制你的第一台设备

对于初次使用的用户，请严格按照以下 4 步完成设备绑定与控制：

### 步骤 1：扫码授权登录
对机器人发送 `/米家登录`。机器人会拉起独立沙盒并返回一个链接，点击打开并使用米家 APP 扫码授权。授权成功后，凭证将安全保存在本地。

### 步骤 2：获取设备 DID 并建立底层档案 (⚠️ 关键)
发送 `/刷新米家`。机器人会从云端极速拉取您账号下绑定的设备列表：
> `1. 【未配置别名】(米家空气净化器 4 Lite) [🟢] (12345678)`
> `2. 【未配置别名】(空调插座) [🟢] (114514)`

**这一步极其重要！** 系统会在后台悄悄把 `12345678` 和官方名称 `米家空气净化器 4 Lite` 绑定。请记录下行尾括号里的这串数字（如 `114514`），这就是设备的 **DID**。

### 步骤 3：在 WebUI 配置好记的别名
*⚠️ 出于安全与精确性考虑，未配置别名的设备将被插件拦截，禁止查看详情与控制。*
前往 AstrBot WebUI 的插件配置页，找到 `device_map` 选项。将刚才的 DID 赋予一个您习惯呼叫的“别名”：
* 键 (Key) [别名]：`"卧室净化器"` 或 `"客厅 空调"` (支持带空格)
* 值 (Value) [DID]：`"114514"`

填写完毕后，保存配置、重载插件并使用`/刷新米家`。

### 步骤 4：享受控制！
此时您可以先发送探测指令查看它当前的工作状态与可用菜单：
> `/米家详情 卧室净化器`

接着，对机器人发送控制指令：
> `/米家控制 卧室净化器 开`
> `/米家控制 卧室净化器 模式 睡眠`

如果听到机器“滴”的一声，恭喜您，您的智能管家正式上线！

## 🎮 指令指南

### 🔑 账号与管理
* `/米家登录`：拉起登录环境获取扫码二维码链接。
* `/米家登出`：安全销毁当前登录状态，清理本地凭证与现场。
* `/米家状态`：查看插件健康度与最近控制记录。

### 📡 设备查询
* `/刷新米家`：极速同步云端设备，输出带 DID 的精简设备总览列表，并**刷新本地物模型缓存**。
* `/米家详情 [设备别名]`：深度嗅探某台具体设备支持的属性调节菜单，并以全中文排版展示设备当前的实况传感器数据。

### 🎛️ 设备控制

**基础电源控制 (强烈推荐)**
* `/米家控制 [设备别名] [开/关]`
    * *示例*：`/米家控制 落地扇 开`

**高级属性控制 (探索功能)**
* `/米家控制 [设备别名] [属性] [值]`
    * *示例*：`/米家控制 空调 温度 26`
    * *示例*：`/米家控制 蒸煮锅 火力 3`
    * *提示：您可以直接使用 `/米家详情` 输出的中文化属性词进行控制，遇到未适配的新功能，系统也完美支持直接使用原生英文底层键值进行盲操透传。*

> 💡 **设备属性未完全适配？欢迎提交 Issue！**
> 如果您在使用探索功能时，发现详情页存在大量未汉化的原生英文属性（或控制不准确）影响了您的使用体验，非常欢迎向我们提交 Issue 以扩充物模型库！
> 
> **提交格式建议：**
> 请直接复制发送 `/米家详情 [设备别名]` 后机器人返回的属性数据，例如：
> ```text
> ✅ 【您的设备名称】支持的高级调整属性:
> status, fault, target_time, target_temperature, left_time, auto_keep_warm, current_keep_warm, mode, recipe_id, recipe_name, recipe_sync, target_cooking_measure, turn_pot, turn_pot_config, texture, reservation_left_time, cooking_weight
> ```
> 我们会在下个版本中为您特有的设备加入专属的中文物模型翻译字典。

## ⚠️ 常见排障 

| 现象 | 可能原因与建议 |
| :--- | :--- |
| **设备找不到 / 共享报错** | 云端可能由于安全策略拒绝了第三方 API 对“共享设备”的非所有者控制权。建议换用设备**主账号**进行扫码登录。 |
| **设备拒绝请求** | 您可能尝试修改了只读属性，或属性值的类型/格式超出该设备允许的范围（可使用 `/米家详情` 确认）。 |
| **被拦截提示需先刷新米家** | 发生了缓存穿透。插件为了精准控制，需要设备的官方名称以匹配图纸字典。请先执行一次 `/刷新米家`。 |
| **网络异常** | 您的宿主机与米家 API 发生 TLS 断连或请求受限，请检查网络环境或稍后再试。 |
